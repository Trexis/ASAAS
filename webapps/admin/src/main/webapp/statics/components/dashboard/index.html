Sites define a web site that should be returned as a service, and also other features like session management, api management, and cache management. Additionally it also define the security strategy to use for authenticating the visitors to the site.
View Sites
System Users are allowed to invoke the managed sites as a service. Note that these are not the site user that authenticate to access the site itself, but instead users granted access to invoke the site as a service. Users are only granted access to their sites.
View System Users
Every request to a site as a service is managed as a session. Session management will reuse the last successful session for subsequent requests to the site. Users are not prompted to provide authentication credentials when this session expire or fails.
